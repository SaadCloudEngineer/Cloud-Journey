## 📚 CHUNK 1 — Internet & Packets
**Status: ✅ Mastered**

### Definition
The Internet is a global network of computers connected together that communicate by sending data in small packets using agreed rules called Protocols.

### Key Concepts
- **Internet** = Global network of connected computers
- **Packet** = Small piece of broken data sent across network
- **Protocol** = Agreed rules for communication
- **Data Flow** = Request sent → Server breaks into packets → Travels through routers → Reassembled at destination

### Real Example
When you search an image on Google:
1. Your phone sends REQUEST through Wi-Fi
2. Request travels through your Router
3. Goes through ISP cables
4. Reaches Google Data Centre
5. Google breaks image into PACKETS
6. Packets travel back through ISP
7. Packets REASSEMBLE at your phone
8. You see the image!

### Commands Practiced
```bash
ping google.com          # Send packets to Google
ping -c 10 google.com    # Send exactly 10 packets
tracert google.com       # See every router (hop) to destination
```

### Lab Results
- Pinged Google, Facebook, YouTube, Amazon
- YouTube server was closest (lowest ms)
- Traced 13 hops to reach Google server
- Saw maa = Mumbai server in tracert!

## 📚 CHUNK 2 — IP Address & DNS
**Status: ✅ Mastered**

### Definitions
- **IP Address** = Unique address for every device on network (like home address)
- **DNS** = Domain Name System — converts google.com → 142.250.x.x (word to number)
- **IPv4** = Old format: 192.168.1.1
- **IPv6** = New format: 2404:6800:4002::200e

### Key Concepts
```
Human types  → google.com
DNS converts → 142.250.143.101
WORD         → NUMBER (not reverse!)
```

### Real Discovery
- My DNS Server = 223.5.5.5 (AliDNS — Alibaba)
- Backup DNS = 223.6.6.6
- Two DNS servers = REDUNDANCY (backup if one fails!)
- Google has 10+ IPs = LOAD BALANCING
- IRCTC server IP = 103.x.x.x = India server 🇮🇳

### Commands Practiced
```bash
nslookup google.com      # Find IP of any website
nslookup facebook.com
nslookup irctc.co.in
ipconfig /all            # See your DNS server (Windows)
ip addr show             # See your IP (Linux)
```

### Lab Results
- Found real IP of Google, Facebook, YouTube, IRCTC
- Bypassed DNS — typed IP directly in browser — it worked!
- Proved: DNS just finds IP, once you have IP you dont need DNS

## 📚 CHUNK 3 — HTTP vs HTTPS & Ports
**Status: ✅ Mastered**

### Definitions
- **HTTP** = HyperText Transfer Protocol — data travels UNENCRYPTED (like postcard)
- **HTTPS** = HTTP Secure — data ENCRYPTED with SSL/TLS (like locked box)
- **Port** = Virtual door number on server (65,535 doors total)
- **SSL Certificate** = Digital lock that enables HTTPS

### Port Numbers
| Port | Service | Security |
|---|---|---|
| 80 | HTTP | ❌ Unsecured |
| 443 | HTTPS | ✅ Encrypted |
| 22 | SSH | ✅ Secure |
| 25 | Email (SMTP) | ⚠️ Basic |
| 3306 | MySQL Database | 🔒 Always block from internet! |
| 53 | DNS | ✅ Internal |

### Key Concepts
```
HTTP  = Postcard — anyone can read
HTTPS = Locked box — only sender/receiver can read

301 Redirect = http → https automatic redirect
SSL Expired  = Browser shows warning — must renew!
```

### Commands Practiced
```bash
netstat -an                    # See all active connections
netstat -an | find "LISTENING" # See open ports (Windows)
ss -tulpn                      # See open ports (Linux)
curl -I https://google.com     # See HTTP response headers
```

### Lab Results
- Saw 8 active HTTPS connections on my laptop
- Google response = 301 (redirect to www)
- Facebook response = 301 (redirect to www)
- Port 53 listening = DNS resolver running
- Clicked SSL certificate — saw valid until date

## 📚 CHUNK 4 — Firewall & Router
**Status: ✅ Mastered**

### Definitions
- **Router** = Traffic director — sends packets on best path to destination
- **Firewall** = Security guard — checks every packet, allows or blocks based on rules

### Key Concepts
```
INBOUND RULES  = Traffic coming INTO your device
                 Blocks hackers trying to reach you

OUTBOUND RULES = Traffic going OUT from your device
                 Controls what your device connects to

FIREWALL RULE TYPES:
✅ ALLOW — trusted IP or Port
❌ BLOCK — unknown or suspicious packet
```

## 📚 CHUNK 6 — Linux Permissions & SSH
**Status: ✅ Mastered**

### Definitions
- **Permissions** = Rules about who can read, write, execute files
- **SSH** = Secure Shell — remotely control any server in world

### Permission System
```
LETTERS:
r = read    (4)
w = write   (2)
x = execute (1)

USERS:
Owner / Group / Others

NUMBERS:
7 = rwx = full access
6 = rw- = read + write
4 = r-- = read only
0 = --- = no access

EXAMPLES:
chmod 700 = rwx------ (owner only) ✅ SAFE
chmod 777 = rwxrwxrwx (everyone)  ❌ DANGEROUS
chmod 644 = rw-r--r-- (normal file)
chmod 000 = --------- (locked out!)
```

### Real Experience
```bash
chmod 000 secret.txt
cat secret.txt
# Permission denied! ❌

chmod 700 secret.txt
cat secret.txt
# Works! ✅
```

### SSH Concept
```
YOUR LAPTOP (India) ──SSH──► AWS SERVER (Mumbai)
Encrypted tunnel
Control server from anywhere in world!
```

### Security Rule
```
chmod 777 = DANGEROUS!
Anyone can read, write, execute
Data leak risk!
Real Cloud Engineers NEVER use 777 on servers!
```
